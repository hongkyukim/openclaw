Major Use Cases & Platform Overview
"The personal AI assistant that never sleeps — and knows where everything is."
Lecture Presentation · 2026
Welcome! Today we explore what OpenClaw actually does — not just what it is. OpenClaw is a personal AI assistant platform that runs locally or on a VPS, connects to your accounts, and acts as your always-on AI companion. We'll cover 10 major use cases, each with real CLI examples.
🧩

What Is OpenClaw?

  • A personal AI assistant platform — runs on macOS, Linux, or a VPS
  • Connects to your real accounts: email, calendar, GitHub, messaging, and more
  • Always-on Gateway daemon — proactive, not just reactive
  • Extensible skill system via clawhub.ai
  • Multi-model: Claude, GPT, Gemini, and more

Local

macOS companion app, instant access, your data stays home

Remote

Headless gateway on VPS/Pi, accessible anywhere via tailnet or reverse proxy
OpenClaw is designed to be YOUR assistant — not a SaaS product that owns your data. It runs where you run it. Skills define capabilities; the gateway daemon keeps it alive.

Personal Productivity Assistant

  • Manage Apple Reminders — add, complete, filter by date or list
  • Create and search Apple Notes and Obsidian vaults
  • Check your calendar for upcoming events, conflicts, and deadlines
  • Proactively surfaces what matters — before you think to ask
  • Works in natural language: "Remind me to call the dentist tomorrow at 9am"
# Agent manages reminders via CLI
remindctl add "Call dentist" --due "2026-04-14 09:00" --list Work
remindctl list --due-before "2026-04-15" --format json
The apple-reminders and apple-notes skills use native macOS CLIs. No cloud sync required — works fully offline on your Mac. Obsidian vaults are plain Markdown, so the agent reads and writes them directly.

Coding Agent Orchestration

  • Delegate complex coding tasks to Claude Code or Codex
  • Build features, fix bugs, refactor — hands-free in the background
  • Review pull requests in isolated temp directories
  • Run multiple agents in parallel with git worktrees
  • Auto-notified when work completes via system events
# Spawn Claude Code on a project (background)
cd ~/Projects/myapp
claude --permission-mode bypassPermissions --print \
  "Add rate limiting to the /api/v1/users endpoint"

# Or via Codex (full-auto, sandboxed)
codex --full-auto "Fix the broken pagination in UserList component"
OpenClaw orchestrates coding agents — it doesn't write the code itself. Claude Code uses --print --permission-mode bypassPermissions for non-interactive runs. Codex needs PTY and a git repo. Parallel PRs reviewed simultaneously with git worktrees.

  • In group chats: participates like a human — reacts, knows when to stay quiet

Proactive Monitoring & Alerts

  • Heartbeat system — periodic background checks (email, calendar, weather)
  • Cron scheduler — exact-time jobs with isolation from main session
  • Checks inbox for urgent messages and surfaces them
  • Calendar alerts when events are within 2 hours
  • Configurable in HEARTBEAT.md — no code required
# Schedule a security audit every Monday at 9am
openclaw cron add \
  --name "healthcheck:security-audit" \
  --schedule "0 9 * * 1" \
  --task "run healthcheck skill"

# One-shot reminder in 20 minutes
openclaw cron add --name "remind-standup" \
  --schedule "+20m" --task "remind me about standup"
Heartbeats vs cron: heartbeats batch multiple checks together at approximate intervals; cron is for exact-time, isolated, one-shot or recurring jobs. HEARTBEAT.md is a plain text checklist — the agent reads it on every wake.

GitHub & DevOps Automation

  • Fetch issues, filter by label/milestone/assignee
  • Spawn coding agents to implement fixes and open PRs
  • Monitor PR review comments — auto-iterate until merged
  • Watch CI runs, surface failures, fetch logs
  • gh-issues skill: full hands-free issue-to-PR pipeline
# Auto-fix open bugs and open PRs
/gh-issues owner/repo --label bug --limit 5 --watch

# Check PR status and CI
gh pr status
gh run list --branch main --limit 5
gh run view 12345 --log-failed
The gh-issues skill is a full orchestration pipeline. It fetches issues, delegates fixes to coding agents, monitors PRs, and responds to review comments — all without manual intervention. --watch mode keeps looping until all issues have merged PRs.

Security & Host Hardening

  • Deep security audit of the OpenClaw host machine
  • Checks: open ports, firewall status, disk encryption, SSH config
  • Risk tolerance profiles: Home Balanced, VPS Hardened, Developer Convenience
  • Step-by-step remediation plan with exact commands
  • Scheduled periodic audits via cron integration
# Run a deep security audit
openclaw security audit --deep

# Apply OpenClaw safe defaults (file permissions, gateway config)
openclaw security audit --fix

# Check for available updates
openclaw update status
The healthcheck skill walks through OS hardening — firewall, SSH, updates, encryption. It never changes host settings without explicit approval. Prefer reversible, staged changes with a rollback plan. Always shows the full plan before executing anything.

Knowledge & Memory System

  • MEMORY.md — curated long-term memory (private, main session only)
  • memory/YYYY-MM-DD.md — daily raw logs of what happened
  • Obsidian vault integration — read, write, and search plain-markdown notes
  • Session log search — semantically query past conversations
  • Agent updates memory autonomously; no manual journaling needed

Short-term

Daily files — raw event logs, decisions, code snippets

Long-term

MEMORY.md — distilled wisdom, preferences, lessons learned
The agent wakes up fresh each session but reads its memory files first. MEMORY.md is only loaded in private main sessions — not in group chats — for security. The session-logs skill uses jq to search indexed conversation history.

Skill Ecosystem (clawhub.ai)

  • Skills are modular, installable capabilities — like npm packages for AI
  • Install new skills on-the-fly: clawhub install <skill>
  • Update to latest or pin a specific version
  • Publish your own skills to share with the community
  • Each skill is a folder with a SKILL.md + optional scripts/references
# Browse and install a skill
clawhub search "slack"
clawhub install slack-notifier

# Update all installed skills
clawhub update --all

# Publish your custom skill
clawhub publish ./my-skill/
apple-notes coding-agent github healthcheck obsidian weather summarize gh-issues tmux taskflow
Skills are the extensibility layer of OpenClaw. Any capability can be packaged as a skill — the SKILL.md tells the agent how to use it. clawhub.ai is the public registry; private skills live in your workspace.

Smart Group Chat Participation

  • Joins Discord servers, Telegram groups, WhatsApp chats as a real participant
  • Knows when not to speak — doesn't flood casual conversation
  • Responds when directly mentioned, asked a question, or can add real value
  • Uses emoji reactions for lightweight acknowledgment (👍 ❤️ 🤔)
  • Never shares private user data in group contexts

The Human Rule

Humans in group chats don't respond to every message. Neither should the assistant. Quality over quantity. If you wouldn't send it in a real group chat — don't send it.
This is a UX philosophy, not just a feature. The agent has access to everything but behaves like a respectful participant. MEMORY.md is never loaded in group chats to prevent private data leakage.

TaskFlow: Durable Multi-Step Jobs

  • Long-running work that spans multiple sessions without losing state
  • Child task linkage — parent job tracks sub-tasks (e.g., one per PR)
  • Waiting states — pause for an API response or human decision, resume automatically
  • Revision-checked mutations — never double-applies a change
  • Powers inbox triage, overnight pipelines, and scheduled report generation
# Example: inbox triage flow
flow: inbox-triage
tasks:
  - id: fetch-emails
    then: classify
  - id: classify
    routes:
      urgent: notify-now
      needs-reply: draft-reply
      newsletter: weekly-summary
TaskFlow is the durable substrate under skills like gh-issues and complex agent pipelines. It handles flow identity, waiting state, and user-facing emergence. Use it when work outlives a single conversation turn.
